사전 작업

'공공 도서관 운영시스템' 이라는 주제를 갖고 해당 홈페이지의 기획안을 구상하였습니다. 좀더 나은 홈페이지를 개발하기 위해 벤치마킹하여 현재 다른 도서관들의 홈페이지의 구성을 비교하여 개발할 수 있도록 준비하였으며, 이에 맞게 요구명세서를 작성하여 각 페이지에 맞는 업무를 구상하였습니다.

이번 Web프로젝트에 Oracle DB를 사용하기로 정하였습니다. 이에 지속적으로 데이터를 CRUD 할 일이 많이 생기기 때문에 각자의 PC에서 개발을 진행하기에는 많은 불편함이 있을 것으로 판단되었습니다. 서버PC를 통해 실시간으로 CRUD를 진행 해야 유동적으로 작업을 진행할 수 있습니다. 허나 서버PC가 따로 없기 때문에 이를 대체할 PC가 필요했고, 제 데스크탑 PC를 서버PC 대체용으로 사용하기로 하였습니다. SQL-Developer에서 DCL을 통해 DB계정을 새로 만들었고, PC 공유기를 포트포워딩 하여 다른 조원들이 DB에 접속 할 수 있도록 환경을 구성하여 프로젝트를 진행하였습니다.

DB 모델링 및 설계

요구 분석서 및 DBMS를 정하고, 요구분석서를 토대로 ERD를 작성하였습니다. ERD는 eXERD 프로그램을 사용하여 진행하였으며 레코드와 컬럼을 구상하여 자료형을 정의해주고, 각각의 테이블을 그려나갔습니다. 그런 다음 테이블들의 관계를 정의하면서 정규화작업을 진행하여 모델링을 완료하였습니다.

완성된 ERD를 기준으로 테이블 설계를 진행했습니다. 식별자를 생성하고, 각각 테이블간의 제약사항들을 걸어 필요한 테이블를 모두 생성하였습니다. 추가로 삭제 할 경우를 대비하여 Drop은 부분을 파일을 따로 생성하였고, 추후 코드 작성 중 추가되는 사항들을 따로 정리하여 최신화 작업을 진행하였습니다.

설계된 DB에 맞는 데이터를 확보하기 위해 OpenAPI를 활용하여 더미데이터를 확보하였습니다. '도서관 정보나루'라는 국립중앙도서관에서 운영하고 있는 OpenAPI를 신청하여, 특정 도서관의 책목록 데이터를 구할 수 있었고, XML파일을 파싱하여 DML을 작성하여 DB에 데이터를 추가하였습니다.

화면설계

개발할 도서관 홈페이지의 전체적인 계층구조를 다이어그램 형태로 작성하였고, Kakao Oven 웹프로그램을 통하여 UI 프로토타입을 작성하였습니다. 추가로 UI구현 표준 구현지침서를 작성하여, 조원들 간의 UI를 통일하기위해 버튼 및 글자색을 맞춰 통일성을 갖출 수 있도록 하였습니다.

코드 작성

전반적으로 Servlet/JSP의 MVC모델 형태로 CRUD형태의 게시판에 페이징 작업과 검색기능을 추가하여 기본적이 게시판을 구현하였습니다.

도서후기이력
회원이 대출한 책에 대해 후기를 남길 수 있는 페이지입니다. Modal을 통해 페이지 이동없이 쉽게 후기를 남길 수 있도록하였습니다. Modal창에는 radio버튼에 이미지를 덮어씌어 별점처리를 할 수 있도록 구현하였고, 이미 작성한 후기가 있을 경우, 수정 또는 삭제를 할 수 있도록 했습니다.

도서기증
도서기증 페이지에서는 회원이 도서관에 책을 기증신청 할 수 있는 페이지입니다. 도서후기 페이지와 비슷하게 구상이 되었으며, 도서기증신청 Modal에서 택배신청시, Modal을 1개 더 띄워 이중으로 구현하였고, 기존 도서기증신청에서 작성한 데이터를 받아와 미리 입력하여 사용자가 책에 관한 정보를 다시 적지 않도록 구현하였습니다.

독서건강분석
독서건강분석 페이지에서는 로그인한 회원의 연령대의 평균 대출 수량 및 대출한 장르를 분석하여 차트로 표시하여 자신의 독서상황을 파악할 수 있도록 지표로 표시해 주었습니다. 차트는 HightChart사에서 제공하는 차트를 사용하여 차트에 맞게 SQL을 통하여 데이터 가공 후 대입하였습니다. 추가로 회원이 가장 많이 대출한 장르를 파악하여 추천도서를 추가하였습니다.

후기

센터에서 교육을 이수하면서 배웠던 언어를 모두 활용할 수 있어 굉장히 만족스러웠던 프로젝트 이면서 익숙하지않은 언어도 많아 걱정이 많았던거 같습니다. 미숙한 언어에 부담감이 있어, 조금 쉬울 수 있지만 무난한 도서관을 주제를 삼은 이유 중에 하나이기도 합니다.

전체적으로 합을 계속 맞춰왔던 팀원들과의 프로젝트 진행이라 팀원들간의 트러블이나 마찰은 없었습니다. 오히려 틈틈히 서로 도와주고, 도움이 될 만한 정보나 공유할 수 있는 클래스, 메소드 등을 공유하면서 소통하느라 바빴던 것같습니다. 프로젝트 마무리 단계에서는 구현을 못 맞친 팀원이 있어 같이 버그를 잡아주면서 마무리할 수 있게 도움도 줄 수 있었습니다.

다만 문제는 자신에게 있었고, Client파트를 구현하는데 있어 어려움이 있었습니다. Java와 SQL이 완벽한건 아니지만 사용함에 있어 거부감이나 불편함은 없었지만, Script쪽 언어를 직접 사용해본적은 많지 않아 시작부터 험난했습니다. 화면 구현을 마치고 데이터를 받아서 표현한다던지, Script쪽에서 데이터를 직접 처리하는 부분에 있어 확실히 미숙한 부분을 느꼈고, 이를 하나하나 다시 학습해 가면서 프로젝트를 진행하다보니 간단한 코드 작성도 시간이 2~3배는 걸렸던거 같습니다.

부족함을 깨달고, 자신의 역량을 향상 시키기위해서는 노력하는 수 밖에 없었고, 다른 프로젝트에 비해 시간을 좀 더 많이 할애하여 프로젝트를 마무리 할 수 있었습니다.

부족한 부분도 많았고, 보강할 부분도 많지만 개인적으로 만족할 수 있었던 프로젝트였습니다.
